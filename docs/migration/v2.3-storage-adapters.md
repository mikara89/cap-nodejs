# v2.3 Storage Adapters Readiness Checklist

v2.3 starts with storage contract hardening and adds the first new first-party
storage adapter, Knex.

Planned adapters:

- Knex available in v2.3: `@mikara89/cap-storage-knex`
- TypeORM planned v2.3: `@mikara89/cap-storage-typeorm`
- Prisma planned v2.3: `@mikara89/cap-storage-prisma`

TypeORM and Prisma are planned only. Do not document them as available until
their implementations, tests, examples, exports, and package dry-runs are
complete.

## Checklist

- Package name follows the `cap-*` rule.
- Package root is framework-free.
- Framework wrappers live under explicit subpaths such as `/nest`.
- Storage passes the publish contract.
- Storage passes the received contract.
- Transaction behavior is documented.
- Rollback behavior is documented.
- Capability model is implemented or explicitly deferred.
- Capabilities are reported conservatively.
- Inbox dedupe uses `group + dedupeKey`.
- Examples are added and compile.
- Package exports are verified.
- Pack dry-run is verified.

## Sequencing

Knex is the first adapter after the foundation PR. TypeORM and Prisma follow
after the shared contracts prove the adapter surface. SQL-core remains deferred
until duplication is proven across real adapters.
