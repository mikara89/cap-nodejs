# v2.3 Storage Adapters Readiness Checklist

v2.3 starts with storage contract hardening and adds new first-party storage
adapters for Knex, TypeORM, and Prisma.

Current v2.3 adapters:

- MikroORM: `@mikara89/cap-storage-mikro-orm`
- Knex: `@mikara89/cap-storage-knex`
- TypeORM: `@mikara89/cap-storage-typeorm`
- Prisma: `@mikara89/cap-storage-prisma`

| Adapter | Adapter style | Transaction context |
| ------- | ------------- | ------------------- |
| MikroORM | ORM-specific adapter | `MikroORM EntityManager` |
| Knex | SQL query-builder adapter | `Knex.Transaction` |
| TypeORM | ORM adapter | `TypeORM EntityManager` |
| Prisma | Raw-SQL Prisma Client adapter; no CAP models in the Prisma schema | `Prisma.TransactionClient` |

Drizzle, Sequelize, and Mongoose remain future candidates. Raw SQL and a shared
SQL core remain deferred until the current adapters prove meaningful
duplication.

## Checklist

- Package name follows the `cap-*` rule.
- Package root is framework-free.
- Framework wrappers live under explicit subpaths such as `/nest`.
- Storage passes the publish contract.
- Storage passes the received contract.
- Transaction behavior is documented.
- Rollback behavior is documented.
- Capability model is implemented or explicitly deferred.
- Capabilities are reported conservatively.
- Inbox dedupe uses `group + dedupeKey`.
- Examples are added and compile.
- Package exports are verified.
- Pack dry-run is verified.

## Sequencing

Knex was the first adapter after the foundation PR, followed by TypeORM and
Prisma. All three use the shared publish and received contracts. SQL-core
remains deferred until duplication is proven across real adapters.
